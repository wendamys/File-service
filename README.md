# File Service Client

Тестовое задание на разработку клиента для API скачивания файлов.

## Описание

Клиент автоматически скачивает все доступные текстовые файлы из сервиса, соблюдая ограничения API.

Алгоритм работы:

1. Получает случайную порцию имён файлов.
2. Разбивает список на группы не более 3 файлов.
3. Скачивает каждую группу ZIP-архивом.
4. Распаковывает архив в директорию `downloads/`.
5. Отмечает файлы как скачанные.
6. Повторяет процесс, пока сервер не вернёт пустой список.

Поддерживается автоматическая обработка ограничения частоты запросов (`429 Too Many Requests`) с использованием заголовка `Retry-After`.

## Установка

Создать виртуальное окружение:

### Windows

```powershell
python -m venv .venv
.\.venv\Scripts\activate
```

### Linux/macOS

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Установить зависимости:

```bash
pip install -r requirements.txt
```

Либо через [uv](https://github.com/astral-sh/uv) (используется в проекте, версия Python зафиксирована в `.python-version`):

```bash
uv sync
```

## Особенности реализации

Проект разделён на несколько независимых компонентов.

### FileServiceClient

Отвечает только за взаимодействие с HTTP API.

Возможности:

- получение списка файлов;
- скачивание файлов;
- отметка скачанных файлов;
- автоматическая обработка ошибок `429`;
- обработка блокировки `403`.

---

### Downloader

Реализует основной сценарий работы приложения.

Последовательно:

- получает список файлов;
- разбивает его на группы по три файла;
- скачивает архивы;
- распаковывает их;
- отмечает файлы скачанными;
- повторяет процесс до полного завершения.

---

### ZipExtractor

Отвечает исключительно за работу с ZIP-архивами.

Получает архив в виде массива байтов и распаковывает его в указанную директорию.

---

## Обработка ошибок

Поддерживаются следующие ситуации:

- автоматическое ожидание при ответе `429 Too Many Requests`;
- обработка временной блокировки (`403 Forbidden`);
- HTTP-ошибки API;
- повреждённые или некорректные ZIP-архивы.

Все ошибки логируются через модуль `logging` (см. `logger.py`).
